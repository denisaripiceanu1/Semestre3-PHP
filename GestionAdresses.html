Gestion de carnet d'adresses avec MySQL

L’objectif est de gérer un carnet d'adresses qui sera sauvegardé dans une base de données. La base doit comporter au moins une table contenant différents champs : nom, prénom, adresse, code postal, ville, téléphone.

  • Ajout d'un contact dans le carnet
  • Recherche d'un contact à partir du nom
  • Modification d'un contact du carnet
  • Suppression d'un contact du carnet
  • Affichage de l'ensemble des contacts du carnet

Saisie d'un contact

Créer un fichier saisie.html permettant de saisir un nouveau contact. Ce fichier contiendra un formulaire constitué des éléments suivants :

  • Un champ de type text correspondant à chacun des éléments à saisir.
  • Un bouton de type reset pour vider les champs du formulaire.
  • Un bouton de type submit pour valider le formulaire.

La validation du formulaire déclenchera l’exécution du script ajoutcontact.php chargé d’enregistrer les informations dans la base tout en s’assurant que le nouveau contact saisi n’est pas déjà présent dans la base.

Recherche de contacts

Proposer un script recherche.php constitué d’un formulaire de saisie d’un ou plusieurs mots-clés, et permettant de retrouver certains contacts enregistrés dans la base. Sur validation du formulaire, ce même script devra afficher, sous la forme d’un tableau HTML, les contacts correspondant aux critères spécifiés par l’utilisateur ainsi que leurs caractéristiques (nom, prénom, adresse, code postal, ville, téléphone).

Aussi, pour chaque contact, associer deux hyperliens dédiés respectivement à la modification et à la suppression du contact.

Effectuer une recherche

Modification d'un contact

L’hyperlien associé à la modification d’un contact doit mener au script modification.php proposant un formulaire similaire à celui réalisé pour l’insertion d’un nouveau contact, mis à part que les champs du formulaire devront être pré-remplis.

Suppression d'un contact

L’hyperlien associé à la suppression d’un contact doit mener au script suppression.php demandant à l’utilisateur la confirmation de la suppression du contact. En cas de validation, le contact correspondant sera supprimé de la base de données et l’utilisateur sera automatiquement redirigé vers la page recherche.php.
